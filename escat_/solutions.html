Our products

Three purpose-built tools for registry co-design, AI-powered data extraction, and data lifecycle governance — built in direct collaboration with clinical and academic partners.

Product 01
Registry Manager

Our interface for co-designing registry variables and evaluating extraction performance with medical experts — built for clinicians who need precision and control without needing to be technical.

Registry Manager is where medical experts and data engineers collaborate. Clinicians define each variable's meaning, acceptable values, validation rules, and the decision tree logic used for extraction — through a guided interface that requires no technical knowledge.

The built-in Schema Field Editor shows field-level validation status across patients on the left, and on the right: the MediXtract output, human output, Chain of Thought reasoning, consistency metrics, and structured discrepancy classification. Experts classify discrepancies as improvements (Filled Blank, Correction, Standardised, Improved Comment) or issues (Missing Documentation, Contradictions, Ambiguous definitions, Structural logic problems). Low consistency scores flag variables for priority review automatically.

Product 02
MediXtract App

A data extraction interface powered by AI-driven contextual reasoning. Surfaces the exact source passage behind every extracted value — guiding accurate, efficient registry population at scale.

The system processes each clinical record through a two-phase pipeline: first generating a structured case summary across 10 critical clinical domains (antibiotics, enteral feeding, respiratory support, infection periods, neurological findings, etc.), then extracting each registry variable using that summary as enriched context alongside the relevant decision tree and Chain of Thought reasoning.

Each variable is run 10 times independently. High consistency means confident extraction. Low consistency triggers automatic flagging for expert review. Every result shows the exact sentence from the source document that drove the extraction — so reviewers verify in seconds, not minutes.

Product 03
DLP

Data Lifecycle Platform — the privacy and governance layer that sits between raw clinical records and the extraction engine, ensuring every record is de-identified, encrypted, and compliant before any AI processing begins.

MediXtract implements a Defense-in-Depth privacy model. Source documents are manually pseudoanonymised first. Then Google Cloud DLP acts as an automated fail-safe — scanning both the visible OCR layer and underlying metadata to detect and redact any residual PII (names, dates, AHV numbers, record IDs). All redactions are burned into the pixel layer so nothing persists beneath cosmetic black bars.

Documents are then translated to English (preserving table structure and spatial layout), reconstructed as searchable PDFs, and only then passed to the extraction engine. All processing is bound to the Zurich region (europe-west6) — Swiss nFADP and GDPR compliant, running on 98% carbon-free energy.
